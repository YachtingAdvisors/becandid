# Be Candid Desktop -- Build Guide

Build, sign, and distribute the Electron desktop app for macOS and Windows.

## Prerequisites

- **Node.js 18+** (22 recommended)
- **npm** (comes with Node.js)
- **macOS** for building macOS targets (`.dmg`, `.zip`)
- **Windows or Linux with Wine** for building Windows targets (`.exe`)

## Install Dependencies

```bash
cd apps/desktop
npm install
```

## Development

```bash
npm run dev
```

This launches the Electron app locally with DevTools available. The app authenticates against the production Be Candid API at `https://becandid.io`.

## Build for macOS

```bash
npm run dist:mac
```

Output: `dist/Be Candid.dmg` and `dist/Be Candid.zip`

### Code Signing and Notarization (macOS)

macOS builds require Apple Developer credentials for signing and notarization. Set the following environment variables before building:

```bash
export APPLE_ID=slaser90@gmail.com
export APPLE_APP_SPECIFIC_PASSWORD=<your-app-specific-password>
export APPLE_TEAM_ID=TH92H9399U
```

**Generate an app-specific password:**

1. Go to https://appleid.apple.com
2. Sign-In and Security > App-Specific Passwords > Generate
3. Use the generated password as `APPLE_APP_SPECIFIC_PASSWORD`

The build pipeline handles signing and notarization automatically:

1. `electron-builder` signs the app with `hardenedRuntime: true`
2. The `afterSign` hook (`scripts/notarize.js`) submits the signed app to Apple's notarization service via `@electron/notarize`
3. Apple validates the app and staples the notarization ticket

If the environment variables are missing, notarization is skipped with a warning. The app will still build but users will see Gatekeeper warnings.

### Entitlements

The macOS build uses `entitlements.mac.plist` which grants:

- Screen capture (`com.apple.security.device.screen-capture`)
- Network access (`com.apple.security.network.client`)
- Apple Events / Accessibility (`com.apple.security.automation.apple-events`)
- JIT and unsigned memory (required for Electron hardened runtime)

## Build for Windows

```bash
npm run dist:win
```

Output: `dist/Be Candid Setup.exe`

Alternatively, use the build script:

```bash
bash scripts/build-windows.sh
```

### Code Signing (Windows)

Windows builds are unsigned by default. To sign with a code signing certificate:

```bash
export CSC_LINK=path/to/cert.pfx
export CSC_KEY_PASSWORD=your-password
npm run dist:win
```

An EV (Extended Validation) certificate eliminates SmartScreen warnings immediately. Standard certificates require building reputation over time. See `docs/WINDOWS-SIGNING.md` for a detailed guide.

## Auto-Update

The app uses `electron-updater` with GitHub Releases as the update provider.

### How it works

1. On launch, the app calls `autoUpdater.checkForUpdatesAndNotify()`
2. `electron-updater` checks the GitHub Releases for the `YachtingAdvisors/becandid` repo
3. If a newer version exists, it downloads and prompts the user to install

### Publishing a release

1. Bump the `version` in `package.json`
2. Build the app: `npm run dist:mac` and/or `npm run dist:win`
3. Create a GitHub Release with the tag matching the version (e.g., `v1.0.1`)
4. Upload the build artifacts:
   - macOS: `Be Candid-{version}.dmg`, `Be Candid-{version}-mac.zip`, `latest-mac.yml`
   - Windows: `Be Candid Setup {version}.exe`, `latest.yml`
5. The YAML files (`latest-mac.yml`, `latest.yml`) are generated by electron-builder and must be included -- they tell the updater what the latest version is

### Auto-publish from electron-builder

You can also publish directly:

```bash
npm run dist -- --publish always
```

This requires a `GH_TOKEN` environment variable with a GitHub personal access token that has `repo` scope.

## Troubleshooting

### "electron-updater is not found"

`electron-updater` must be in `dependencies` (not `devDependencies`) because it runs at runtime inside the packaged app. Run `npm install` to ensure it is installed.

### macOS: "Be Candid can't be opened because Apple cannot check it for malicious software"

The build was not notarized. Ensure `APPLE_ID`, `APPLE_APP_SPECIFIC_PASSWORD`, and `APPLE_TEAM_ID` are set before building. Check the build output for "Notarization complete!" or any notarization errors.

### macOS: Screen recording permission not granted

On first launch, macOS prompts for screen recording permission. If denied, the app cannot capture screenshots. Users must grant permission in System Settings > Privacy & Security > Screen Recording.

### Windows: SmartScreen warning

This appears because the build is unsigned or uses a standard (non-EV) certificate. See `docs/WINDOWS-SIGNING.md` for signing instructions. Users can bypass SmartScreen by clicking "More info" > "Run anyway".

### Build fails with "icon not found"

Ensure all icon files exist in `assets/`:
- `icon.icns` for macOS
- `icon.ico` for Windows
- `tray-icon*.png` files for the system tray

### "Cannot find module 'electron-updater'"

This usually means `electron-updater` was installed as a devDependency instead of a regular dependency. It must be in `dependencies` in `package.json` so it gets bundled into the app.

### Auto-update not working

1. Check that the `publish` config in `electron-builder.yml` points to the correct GitHub repo
2. Verify the GitHub Release tag matches the version format expected by electron-updater
3. Ensure `latest-mac.yml` or `latest.yml` is uploaded alongside the installer in the GitHub Release
4. Check the app logs for updater errors (on macOS: `~/Library/Logs/Be Candid/`)

### Build hangs or is very slow

Electron builds download platform-specific binaries. Ensure you have a stable internet connection. The first build takes longer due to Electron binary downloads.
